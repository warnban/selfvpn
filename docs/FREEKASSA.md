# Freekassa — настройка автооплаты в веб-кабинете

Домен проекта: **https://daddyvpn.site**  
VPS: `72.56.124.163`

Интеграция через **API** (`https://api.fk.life/v1/`), не через SCI-форму.

## 1. DNS (в панели регистратора домена)

| Тип | Имя | Значение |
|-----|-----|----------|
| A | `@` | `72.56.124.163` |
| A | `www` | `72.56.124.163` |

Подождите 5–30 минут, пока DNS обновится.

## 2. HTTPS на сервере

```bash
ssh root@72.56.124.163
cd /opt/selfvpn
chmod +x deploy/setup-domain.sh
bash deploy/setup-domain.sh
```

Или вручную — конфиг: `deploy/nginx-daddyvpn.site.conf`

## 3. `.env` на сервере

```env
WEB_BASE_URL=https://daddyvpn.site

FREEKASSA_MERCHANT_ID=ваш_shop_id
FREEKASSA_API_KEY=ключ_из_личного_кабинета
FREEKASSA_SECRET_2=секретное-слово-2
# Опционально, если nginx не передаёт реальный IP клиента
FREEKASSA_CLIENT_IP_FALLBACK=72.56.124.163
```

API ключ берётся в личном кабинете Freekassa → настройки.

```bash
cd /opt/selfvpn && docker compose restart
```

## Что вписать в форму Freekassa

| Поле | Значение | Метод |
|------|----------|-------|
| **URL сайта** | `https://daddyvpn.site` | — |
| **Секретное слово 2** | → `FREEKASSA_SECRET_2` в `.env` | — |
| **URL оповещения** | `https://daddyvpn.site/payment/notify` | **GET** |
| **URL успешной оплаты** | `https://daddyvpn.site/payment/success` | GET |
| **URL неудачи** | `https://daddyvpn.site/payment/fail` | GET |

Секретное слово 1 (SCI) для API не используется.

## Способы оплаты (параметр `i` в API)

| `i` | Способ |
|-----|--------|
| `44` | СБП (НСПК) |
| `36` | Банковские карты РФ |
| `12` | Банковская карта МИР |

В кабинете пользователь выбирает способ перед оплатой.

---

## Как это работает

1. Пользователь в веб-кабинете → «Пополнить» → срок и способ (СБП или карта) → «Оплатить»
2. Сервер создаёт заказ через `POST /orders/create` и редиректит на ссылку из поля `location`
3. После оплаты Freekassa шлёт webhook на `/payment/notify` → баланс зачисляется → ответ `YES`
4. Пользователь возвращается на `/payment/success` → кабинет с сообщением «Баланс пополнен»

**Email** в API: реальный email пользователя или `{telegram_id}@telegram.org`.  
**IP** в API: реальный IP клиента (из `X-Real-IP` / `X-Forwarded-For`). `127.0.0.1` блокируется — при локальном тесте нужен fallback IP.

Оплата в **Telegram-боте** по-прежнему через Stars или ручную модерацию.

## Проверка

1. В Freekassa нажмите **«Сохранить и проверить»** — notify URL не должен отдавать 404
2. Включите **тестовый режим** и сделайте тестовый платёж из кабинета
3. Логи: `docker compose logs -f web`

---

## Частые проблемы

| Симптом | Решение |
|---------|---------|
| «Сохранить и проверить» не проходит | Нет HTTPS или домен не указывает на VPS |
| Баланс не зачислился | Проверьте `FREEKASSA_SECRET_2`, метод GET, логи web |
| `wrong sign` в логах | Секретное слово 2 не совпадает с личным кабинетом |
| `hacking attempt!` | Запрос не с IP Freekassa (проверяйте через реальный платёж) |
| Ошибка создания заказа | Проверьте `FREEKASSA_API_KEY`, `FREEKASSA_MERCHANT_ID`, IP клиента |
| `nonce` ошибка | Перезапуск с тем же nonce — редко; обычно помогает повтор через пару секунд |
